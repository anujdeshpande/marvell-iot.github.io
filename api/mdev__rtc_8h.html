Detailed Description

RTC driver.

RTC (Real Time Clock) driver provides mdev interface to access the internal RTC. Follow the steps below to use RTC driver from applications

  1. RTC driver initialization

Call rtc_drv_init from your application initialization function. This will initialize and register rtc driver with the mdev interface.

  1. Open RTC device to use

rtc_drv_open should be called once before using RTC hardware for communication.

{
mdev_t *rtc_dev;
rtc_dev = rtc_drv_open("MDEV_RTC");
}
  1. Set RTC configuration

Call rtc_drv_set from your application to set desired upper count The RTC runs at 1KHz by default (for A0 version only).

  1. Set RTC callback function

Call rtc_drv_set_cb from your application to register callback handler with RTC driver. This will be called after clock exceeds the upper value specified in step 3, until RTC is explicitly stopped using rtc_drv_stop. While RTC continues to run , this will be called everytime the counter value wraps around. RTC can be stopped by calling rtc_drv_stop.

  1. Reset the RTC counter

Call rtc_drv_reset to reset the rtc counter before starting .

  1. Start RTC operation

Call rtc_drv_start to start rtc.

  1. Get RTC counter value

Call rtc_drv_get to rtc counter value.

  1. Stop RTC operation

Call rtc_drv_stop to stop rtc.

  1. Closing a device.

A call to rtc_drv_close informs the RTC driver to release the resources that it is holding.

Note
This must not be called from callback registered in step 4 or any interrupt context.

Typical usecase:

rtc_drv_close(rtc_dev);
