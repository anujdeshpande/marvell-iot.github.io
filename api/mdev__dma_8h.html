Detailed Description

DMA Module.

mdev_dma is a module that handles DMA driver functionality a layer above the low-level driver code. It establishes a mechanism to deploy DMA capabilities to all peripherals, possibly simultaneously.

The DMAC hardware supports 32 DMA channels. By default, four channels can operate simultaneously. User may configure the value of NUM_DMA_CHANNELS to modify the number of supported channels.

This module provides a set of APIs to use DMA. DMA transfer can be used for the following modes.

  1. MEMORY_TO_PERIPHERAL
  2. PERIPHERAL_TO_MEMORY
  3. MEMORY_TO_MEMORT
    Note
    PERIPHERAL_TO_PERIPHERAL transfer is not supported.
    DMA driver limitations
  1. DMAC supports a maximum of 8191 byte data transfer with the added restriction that SOURCE and DESTINATION ADDRESSES should align with the transfer width.

  2. DMA transfers will be asynchronous.

    A typical DMA usage scenario is as follows:

    1. Initialize the DMA driver with a call dma_drv_init()
    2. Open a channel for DMA transfer using dma_drv_open()
    3. User needs to configure dma_config_type.
    4. [Optional] Call dma_drv_set_cb to set callback for a specific dma channel.
    5. Call dma_drv_transfer dma_drv_transfer with appropriate parameters.
    6. Close the DMM channel, on transfer complete.
    7. de-initialize DMA driver provided no code is using DMA.

int dma_drv_close ( mdev_t *  mdev_dma)

Close DMA channel.

This API closes the DMA channel specified by the given handle. The resources associated with the channel are freed up. This channel now can be selected by dma_drv_open for another DMA transfer. This API also de-registers the registered callbacks on the channel.
Parameters
[in]mdev_dmahandle to DMA channel
Returns
WM_SUCCESS if channel is closed successfully
-WM_E_INVAL if the parameters are invalid

De-initialize DMA driver.

mdev_t* dma_drv_open ( )

Open a dma channel.

This API selects a channel for DMA transfer. It will select a free channel and reserve the same till dma_drv_close() is explicitly called.
Precondition
dma_drv_init()
Returns
handle to dma channel. It is channel number type-casted to pointer of type mdev_t.
-NULL in case the API fails to select a DMA channel
Note
Any available channel can be used for data transfer There is no mapping between DMA channel and peripheral. However correct handshaking interface must be chosen during memory to/from peripheral data transfer.
